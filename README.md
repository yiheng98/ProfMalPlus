## Overview

ProfMalPlus is a malicious npm package detector that combines behavior-graph analysis with coordinated LLM-based reasoning over evidence-enriched code slices. It focuses on installation-time and import-time behaviors by first identifying installation commands and entry files, then constructing object-sensitive behavior graphs that capture sensitive APIs, third-party calls, and runtime-dependent or unresolved calls.

Based on these graphs, ProfMalPlus extracts security-relevant code slices and augments them with evidence from static analysis, third-party package metadata, and sandboxed dynamic execution. Local judge agents reason over individual slices, a global judge agent synthesizes their results into entry-level verdicts, and a localization agent reports concrete malicious code snippets with explanations.

## Getting started

### Prerequisites

The detector runs only on Linux. Docker is required for dynamic analysis.

- Ubuntu 22.04
- [Docker](https://www.docker.com/)
- Java/JDK for [Joern](https://docs.joern.io/installation/). Joern requires a Java environment; see the Joern installation documentation for detailed environment setup.

### Setup

- This implementation targets **Python 3.11**. Install dependencies with:

  ```shell
  pip install -r requirements.txt
  ```

- The CPG is generated by [Joern](https://joern.io/). We provide a customized Joern CLI at https://drive.google.com/file/d/1g-nJbcdgh13x8XKeF8Q4XuSo0i7Z8P5y/view?usp=sharing, which can be used directly.

- Call graphs are generated by [Jelly](https://github.com/cs-au-dk/jelly) 0.10.0. Install Jelly with:

  ```bash
  npm install -g @cs-au-dk/jelly
  ```

  Alternatively, if you already have another Jelly version installed, you can run `jelly/lib/main.js` in `jelly_helper.py`. Please refer to [Jelly](https://github.com/cs-au-dk/jelly) for more details.

- Set up `config.yaml`:

  ```yaml
  joern_path: "./joern-cli-4.0.227/joern-cli" # Path to the Joern CLI.

  LLM:
    model: "deepseek-v4-flash"
    api_key: "your_api_key"
    base_url: "https://api.deepseek.com" # Any OpenAI-compatible API endpoint.
  ```

  - Set `joern_path` to the Joern CLI we provide. Make sure the path is correct if you already have a global Joern installation.
  - Set the LLM model name and API key used to access the model.
  - Set `base_url` to the API endpoint. You can change it to any endpoint compatible with the OpenAI API.

## Usage

Run `profmalplus.py` to analyze a single package:

```bash
python profmalplus.py \
  --package_path "<path-to-package>" \
  --workspace_path "<path-to-workspace>" \
  --dynamic_support true
```

- `--package_path` (required): The path to the target package. The package name is derived from the last component of this path. Please ensure the package is unzipped before analysis.

  - The package to be analyzed should have the following structure:

    ```
    <package-name>/
    └── package/            # REQUIRED subdirectory
        ├── package.json
        ├── code files      # source code
        └── ...
    ```

- `--workspace_path` (optional): The directory where Joern and Jelly write their analysis results and where the behavior graph is saved. Defaults to a `profMalPlus_workspace` directory created two levels above `profmalplus.py`.

- `--dynamic_support` (optional, default `true`): If `true`, dynamic analysis is included.

- `--verbose` (optional, default `false`): If `true`, detailed node-identification logs from the call-processing path are shown.

## Output

After detection, ProfMalPlus writes the results to the workspace of the analyzed package.

For a package named `<package_name>`, the main outputs are saved under `<workspace_path>/<package_name>/`:

- `<workspace_path>/<package_name>/report.json`: the package-level detection report.
- `<workspace_path>/<package_name>/localization/`: detailed malicious-code localization results for flagged files.
- `<workspace_path>/<package_name>/work_flow.log`: the runtime log.

### report.json

`report.json` summarizes the final detection result for the package. Its key fields are:

- `package_name`: The name of the analyzed package.
- `maliciousness_in_package_json`: Whether malicious content is detected in `package.json`.
- `install_time_script`: The install-time scripts found in `package.json`, and whether they contain malicious shell commands.
- `entry_final_verdicts`: A list of per-entry-file results. Each item records an entry file and the detection result of that file.
- `overall_statuses`: The final verdict for the whole package.

### localization/

The `localization/` directory contains detailed code localization results for each file flagged as malicious. Each entry records the `entry` file, a `summary` of the malicious behavior, and a list of `locations`, where each location gives the specific `file` and the responsible `code` snippet.
